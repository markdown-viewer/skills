# Fault Tree Analysis

A fault tree analysis (FTA) diagram using AND/OR logic gates to decompose a top-level system failure into root causes.

Based on template: `engineering/fault_tree_2.drawio`

## Stencils Used

| Component | Stencil | Role |
|-----------|---------|------|
| AND Gate | `mxgraph.electrical.logic_gates.and` | All inputs required for fault |
| OR Gate | `mxgraph.electrical.logic_gates.or` | Any input sufficient for fault |
| Top Event | `rectangle` (rounded=0) | System-level failure |
| Condition | `rectangle` (rounded=1) | Intermediate fault event |
| Basic Event | `ellipse` | Leaf-level root cause (e.g., sensor failure) |
| Transfer | `rhombus` | Reference to sub-tree |

- **All shapes**: `fillColor=#ffffff;strokeColor=#000000;strokeWidth=1`
- **Gates**: `rotation=-90` (rotated to point upward in top-down tree)
- **Connections**: `edgeStyle=orthogonalEdgeStyle;endArrow=none` (plain lines, no arrows)
- **Gate inputs**: connect to `entryX=0;entryY=0.165` (left input) and `entryX=0;entryY=0.835` (right input) entry points

## Pattern Notes

1. **Top-down hierarchy**: Top event (Result) at top → gates → intermediate conditions → basic events at bottom
2. **Gate rotation**: Logic gates use `rotation=-90` to point upward in the tree structure
3. **Gate entry points**: Left input at `entryY=0.165`, right input at `entryY=0.835` (perimeter connection points)
4. **OR gate** (`logic_gates.or`): output fires if ANY input is true — used for alternative failure modes
5. **AND gate** (`logic_gates.and`): output fires only if ALL inputs are true — used for combined failure conditions
6. **Node shapes**: Rectangle = top/intermediate event, Rounded rectangle = condition, Ellipse = basic event (sensor/component), Rhombus = transfer symbol
7. **Monochrome style**: All elements use `fillColor=#ffffff;strokeColor=#000000` — standard for FTA diagrams
8. **No arrows**: All edges use `endArrow=none` — tree structure implies direction top-to-bottom
