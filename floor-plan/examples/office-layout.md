# Office Layout Floor Plan

An L-shaped open office with reception lounge, meeting rooms, kitchen area, workstation rows, private office, and large conference room.

Based on templates: `plans/floor_plan_2.drawio`, `plans/floor_plan_4.drawio`, `plans/floor_plan_5.drawio`

## Key Elements

| Component | Shape / Stencil | fillColor | strokeColor |
|-----------|----------------|-----------|-------------|
| Wall (straight) | `mxgraph.floorplan.wall` | `#000000` | — |
| Wall (L-corner) | `mxgraph.floorplan.wallCorner` | `#000000` | — |
| Door (double) | `mxgraph.floorplan.doorDouble` | `#FFFFFF` | — |
| Door (left swing) | `mxgraph.floorplan.doorLeft` | `#FFFFFF` | — |
| Window | `mxgraph.floorplan.window` | `#ffffff` | — |

| Chair | `mxgraph.floorplan.chair` | `#ffffff` | `#000000` |

| Refrigerator | `mxgraph.floorplan.refrigerator` | `#FFFFFF` | — |
| Dresser / Credenza | `mxgraph.floorplan.dresser` | `#FFFFFF` | — |
| Plant | `mxgraph.floorplan.plant` | `#FFFFFF` | — |

- **Wall style**: `shape=mxgraph.floorplan.wall;fillColor=#000000` — 10px thick. Use `direction=south` for vertical walls. Use `wallCorner` for L-shaped junctions; specify `direction` to select corner orientation (default=┌, `south`=┐, `north`=└, `west`=┘)
- **Door style**: `shape=mxgraph.floorplan.doorLeft;fillColor=#FFFFFF` — single door 80×85, double door (`doorDouble`) 160×85. Split wall into segments to leave a gap where the door is placed
- **Window style**: `shape=mxgraph.floorplan.window;fillColor=#ffffff` — 10px in thin dimension, placed overlapping the exterior wall. Use `direction=south` for windows on vertical walls
- **Workstation group**: A `group;rotation=180` containing: desk (`rect` 80×50) + workstation monitor (`mxgraph.floorplan.workstation` 50×40, `flipV=1`) + office chair (`mxgraph.floorplan.office_chair` 40×43). Child coordinates are relative to the group
- **Furniture**: All use `fillColor=#FFFFFF` (white fill) consistent with floorplan stencil conventions. Basic `rect` shapes serve as desks, countertops, and filing cabinets

## Pattern Notes

1. **wallCorner builds the perimeter** — use `wallCorner` to form L-shaped junctions at each turn of the building outline. Specify `direction` to select corner orientation: default=┌ (top+left edges), `south`=┐ (top+right), `north`=└ (bottom+left), `west`=┘ (bottom+right). Adjacent corners must overlap by ≥10px for seamless connection
2. **Doors require wall gaps** — split a wall into segments leaving a gap equal to the door width (80px for single, 160px for double). Place the door shape in the gap. Use `flipV=1` to flip the swing arc, `direction=north/south` for doors on vertical walls
3. **Windows overlay walls directly** — place `mxgraph.floorplan.window` (10px thick, `fillColor=#ffffff`) on top of the exterior wall at the same position. The white fill visually replaces the black wall. Use `direction=south` for windows on vertical walls
4. **Workstation group pattern** — each workstation is a `group;rotation=180` containing 3 child elements with relative coordinates: a desk (`rect` 80×50 at origin), a monitor (`workstation` 50×40, `flipV=1` at x=15), and a chair (`office_chair` 40×43 at x=20,y=27). All children also carry `rotation=180` to counteract the group rotation on their visual appearance
5. **Cubicle dividers between workstations** — a vertical wall segment (`wall;direction=south` 10×140) acts as a structural partition between adjacent desks. A `rect` (50×130) filing cabinet sits beside each wall divider. This creates the classic cubicle row pattern
6. **Kitchen counter layout** — kitchen appliances are arranged along a wall: `refrigerator` → countertop (`rect`) → `sink_double` → countertop → `range_1` → countertop. All use `direction=south` when placed against a vertical wall
7. **Conference table with chairs** — use an `ellipse` for the table with `chair` shapes arranged around it (2 at each end, 1 on each long side). Use `direction` and `rotation` on each chair to face the table center. For larger tables, use more chairs on the long sides
8. **All furniture uses white fill** — `fillColor=#FFFFFF` with `strokeColor=#000000`. This is the universal convention for draw.io floorplan stencils. Basic `rect` shapes serve multiple roles (desk, counter, filing cabinet) distinguished only by size and placement context
9. **Spatial diagram** — floor plans are purely spatial with no connecting edges. Furniture shapes are positioned within room boundaries defined by walls. No floor/background fills — wall structure alone defines room boundaries
