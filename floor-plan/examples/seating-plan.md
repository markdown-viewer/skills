# Event Venue Floor Plan

A banquet hall floor plan with commercial kitchen, main dining hall, private dining rooms, lounge area, restrooms, and staircase.

Based on template: `plans/seating_plan_1.drawio`

| Component | Shape / Stencil | fillColor | strokeColor |
|-----------|----------------|-----------|-------------|
| Wall (straight) | `mxgraph.floorplan.wall` | `#000000` | — |

- **Wall style**: `shape=mxgraph.floorplan.wall;fillColor=#000000` — 10px thick. Use `direction=south` for vertical walls. Use `wallCorner` for L-shaped junctions
- **Door style**: `shape=mxgraph.floorplan.doorLeft;fillColor=#FFFFFF` — single door 80×85, double door (`doorDouble`) 160×85. Split wall into segments to leave a gap where the door is placed
- **Window style**: `shape=mxgraph.floorplan.window;fillColor=#ffffff` — 10px in thin dimension, placed overlapping the exterior wall
- **Furniture**: All use `fillColor=#FFFFFF` (white fill) consistent with floorplan stencil conventions

## Pattern Notes

1. **Round table group** — a `group` containing an `ellipse` (round table) with `chair` shapes arranged around it. Large tables (160×160) seat 8 chairs at cardinal + diagonal positions using `direction` (north/south/west) and `rotation` (-45°, 45°, 135°, -135°). Small tables (100×100) seat 4 chairs at diagonal positions using `rotation=45` with `direction` for each quadrant
2. **Rectangular banquet table group** — a `group;rotation=90` containing a `rect` (160×100) table with 6 `chair` shapes: 2 on each long side (`rotation=90`) + 1 at each short end (`direction=north/south;rotation=90`). Rotate the entire group with `rotation=90` to orient tables vertically
3. **Lounge seating group** — a `group` containing 2 `couch` shapes facing each other (`direction=north` + `direction=south`) with a `rect` center table between them
4. **Commercial kitchen layout** — refrigerators grouped in pairs (`flipH=1` on one to create double-door effect), `range_2` stoves, `dresser` buffet counters, `sink_double` sinks, and `corner` (L-shaped counter). Appliances use `direction` to orient against walls
5. **Restroom stalls** — repeated pattern of `wall` segments creating partitions, `doorRight;direction=north` for stall doors, and `toilet;direction=south` inside each stall. `sink_2` sinks arranged in a row along the opposite wall
6. **wallCorner builds the perimeter** — use `wallCorner` for L-shaped junctions at each turn of the building outline. Specify `direction` to select corner orientation (default=┌, `south`=┐, `north`=└, `west`=┘). Adjacent corners must overlap by ≥10px for seamless connection
7. **Decorative plants as space fillers** — `plant` shapes placed at corners, beside entrances, and between functional areas to fill dead space and add visual interest
8. **All furniture uses white fill** — `fillColor=#FFFFFF` with `strokeColor=#000000`. Basic shapes (`rect`, `ellipse`, `corner`) serve as tables, counters, and surfaces distinguished by size and context
9. **Spatial diagram** — floor plans are purely spatial with no connecting edges. Furniture shapes are positioned within room boundaries defined by walls. No floor/background fills — wall structure alone defines room boundaries
